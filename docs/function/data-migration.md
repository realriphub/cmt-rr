# 数据迁移

目前支持迁移数据的评论框架有以下，请保证导出的格式为 json 格式。

- **CWD**（CWD 自身导出的评论 JSON，可用于备份与恢复）
- **Twikoo**
- **Artalk**（如果是 .artalk 后缀，导出到本地后修改后缀为 .json 即可）

其他评论框架逐步支持，欢迎提交反馈，可以将你想要迁移进来的框架以及相关结构提交到评论区，我会尽快添加支持。

---

**使用方法：**

`后台管理 > 数据管理 > 评论数据` 区块下：

- 使用「导出 JSON」按钮，可导出当前所有评论数据（CWD 原生格式），用于备份或在其他环境恢复；
- 使用「导入评论」按钮，先在下拉框中选择要导入的框架类型（CWD / Twikoo / Artalk），再选择本地 JSON 文件进行导入。

迁移程序会自动判断是否包含域名前缀（因部分框架默认不输出域名前缀），然后可以自定义前缀。

---

**其他数据导入导出：**

在 `后台管理 > 数据管理` 页面，还提供以下操作：

- **系统配置**：导出 / 导入 Settings 表中的配置（评论设置、邮件设置、功能开关等）；
- **访问统计**：导出 / 导入页面访问量、按日访问和点赞明细；
- **全量备份**：一键导出或导入「评论 + 配置 + 统计」全部数据，适合整站迁移或备份恢复。

具体接口说明可参考「管理员 API / 数据管理相关」章节。
